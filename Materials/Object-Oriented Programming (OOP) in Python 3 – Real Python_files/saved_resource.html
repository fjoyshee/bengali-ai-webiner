                                                                                                                          Florian Dahlitz Matthew DeWitt • 4 months ago

                                                                                                                          The number in the braces indicate the position of the argument in the format method. Using the provided example and swapping the indices would result in the following:


                                                                                                                          if philo.species == "mammal":
                                                                                                                          print("{1} is a {0}!".format(philo.name, philo.species))
                                                                                                                          # mammal is a Philo!

                                                                                                                          The idea is, that you can change the sentence afterwards without touching the argument list. I hope this answers your question. If you want to know more about the topic, let me know.

                                                                                                                                                                    Fanchen Bao —

                                                                                                                                                                    Wonderful article. It really gets me thinking about what objects can have inheritance relationship. It is explained in the article that square should not inherit from rectangle. Another way to see this is that square has all the properties rectangle has: num_sides, side_lengths, interior_angles. The only difference is that for square, all side_lengths are the same. Therefore, square is not a new concept based on rectangle. It is rectangle with a different property.

                                                                                                                                                                    Following this thought process, can we even say rectangle is an inheritance of polygon? A polygon has the same properties as rectangle, with the only difference being that rectangle has 4 sides and the interior_angles are all 90 degrees. In other words, all 2D shapes with straight sides are just polygons with specific properties, and thus there should NOT be any inheritance relationship among them.

                                                                                                                                                                    However, polygon can be inherited from 2D_shape, because polygon adds NEW properties to 2D_shape: straight sides and interior angles. This property is only with polygon but not 2D_shape. Similarly curved_shape can be inherited from 2D_shape, yet circle and eclipse probably should not be inherited from curved_shape.

                                                                                                                                                                    It seems to me that too liberal a mindset of creating inheritance could be a cause of class explosion. It also seems to me that some instances of inheritance that we usually think of (e.g. Employee -> SalaryEmployee, Car -> SportsCar, Dog -> Husky, etc.) probably can be better implemented with the mindset of policy-based design.
